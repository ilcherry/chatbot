## CSS 样式对比

### 机器人消息样式

```css
.message.bot {
  display: flex;
  gap: 8px;
  margin-bottom: 16px;
}

.message-bubble {
  background: white;
  padding: 10px 14px;
  border-radius: 12px;
  border-bottom-left-radius: 4px;
}
```

### 系统消息样式

```css
.message.system {
  display: flex;
  justify-content: center;  /* 居中对齐 */
  align-items: center;
  margin: 12px 0;
}

.system-message-text {
  background: rgba(0, 0, 0, 0.05);  /* 浅灰背景 */
  color: #666;                       /* 灰色文字 */
  padding: 6px 12px;
  border-radius: 12px;
  font-size: 12px;                   /* 较小字体 */
  text-align: center;
}
```

### 用户消息样式

```css
.message.user {
  display: flex;
  flex-direction: row-reverse;  /* 右对齐 */
  gap: 8px;
  margin-bottom: 16px;
}

.message.user .message-bubble {
  background: #5b6fd8;  /* 主题色背景 */
  color: white;
  border-bottom-right-radius: 4px;
}
```

## 设计特点

### 视觉层次

1. **机器人消息** - 白色背景 + 头像，最醒目
2. **用户消息** - 彩色背景，次醒目
3. **系统消息** - 灰色背景，最低调

### 布局策略

1. **机器人消息** - 左侧对齐，展示对话主导者
2. **用户消息** - 右侧对齐，区分用户输入
3. **系统消息** - 居中对齐，独立于对话流程

### 字体大小

| 消息类型 | 字体大小 | 用途 |
|---------|---------|------|
| 机器人消息 | 14px | 标准对话文本 |
| 用户消息 | 14px | 标准对话文本 |
| 系统消息 | 12px | 降低视觉干扰 |

### 颜色方案

| 元素 | 颜色 | 用途 |
|------|------|------|
| 机器人气泡 | `#ffffff` (白色) | 清晰展示回复 |
| 用户气泡 | `#5b6fd8` (主题色) | 区分用户输入 |
| 系统背景 | `rgba(0,0,0,0.05)` | 低调显示通知 |
| 系统文字 | `#666666` (灰色) | 降低视觉权重 |
| 时间戳 | `#999999` (浅灰) | 辅助信息 |

## 动画效果

### 淡入动画

所有消息（包括系统消息）都使用相同的淡入动画：

```css
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(10px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.message {
  animation: fadeIn 0.3s ease;
}
```

## 可访问性考虑

1. **对比度** - 系统消息文字颜色 `#666` 在 `rgba(0,0,0,0.05)` 背景上对比度 > 4.5:1
2. **字体大小** - 最小 12px，确保可读性
3. **间距** - 消息间距 12px，避免拥挤
4. **语义化** - 使用适当的 HTML 结构和 class 名称

## 最佳实践

### 长度控制

```typescript
// ✅ 好的示例 - 简短明了
"🟢 客服已上线"
"💾 会话已保存"

// ❌ 避免 - 过长的系统消息
"系统提示：您的会话已经成功保存到服务器，所有的对话记录将被永久保留"
```

### 频率控制

```typescript
// ✅ 好的示例 - 关键状态变化
客服上线 → 显示系统消息
转接人工 → 显示系统消息

// ❌ 避免 - 过于频繁
每条消息都显示"已读"系统提示
每隔几秒显示心跳消息
```

### 内容质量

```typescript
// ✅ 好的示例 - 有价值的信息
"⚡ 平均响应时间: 2秒"
"🔄 正在为您转接人工客服..."

// ❌ 避免 - 无意义的信息
"系统正在运行"
"消息已发送"（用户已经看到了）
```
